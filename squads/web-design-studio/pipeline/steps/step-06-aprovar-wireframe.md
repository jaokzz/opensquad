# Step 06: Aprovar Wireframe

O wireframe está pronto em `output/wireframes.md` — a estrutura, ordem e proporção de cada seção, ainda sem cor ou foto.

Aprovar aqui evita retrabalho caro na fase de mockup em alta-fidelidade, onde cada mudança de estrutura custa mais tempo.

Revise:
1. A ordem das seções faz sentido para levar o visitante até a ação desejada (agendar, comprar, contatar)?
2. Falta alguma seção importante do negócio, ou sobra alguma que não agrega?
3. Os pontos de assimetria (wordmark, sangria de foto, zig-zag) fazem sentido para esse cliente específico?

Aprova como está, ou quer ajustar a estrutura antes de seguir para o mockup?
